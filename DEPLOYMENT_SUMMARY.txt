================================================================================
DEPLOYMENT STEPS (STREAMLIT CLOUD):
================================================================================

STEP 1: PREPARE YOUR CODE
-------------------------
1. Make sure you have:
   - email_agent_streamlit_cloud.py (or use original with modifications)
   - send_mails_gmail_api_cloud.py
   - requirements.txt (with all dependencies)
   - .streamlit/config.toml
   - .gitignore

2. Update requirements.txt:
   streamlit>=1.28.0
   google-auth>=2.0.0
   google-auth-oauthlib>=0.5.0
   google-api-python-client>=2.0.0
   dnspython>=2.4.0

STEP 2: PUSH TO GITHUB
----------------------
1. Create GitHub repository (if not exists)
2. Push your code:
   git add .
   git commit -m "Prepare for cloud deployment"
   git push origin main

IMPORTANT: Never commit credentials.json or token.json!

STEP 3: SETUP GOOGLE CLOUD OAUTH
---------------------------------
1. Go to Google Cloud Console
2. APIs & Services > Credentials
3. Edit your OAuth 2.0 Client
4. Add Authorized redirect URIs:
   - For Streamlit Cloud: https://your-app-name.streamlit.app
   - For local: http://localhost:8501
5. Save changes

STEP 4: DEPLOY ON STREAMLIT CLOUD
----------------------------------
1. Go to https://streamlit.io/cloud
2. Sign in with GitHub
3. Click "New app"
4. Configure:
   - Repository: Select your repo
   - Branch: main
   - Main file path: email_agent_streamlit_cloud.py
   - App URL: Choose name (e.g., ai-email-agent)
5. Click "Advanced settings"
6. Go to "Secrets" tab
7. Add secrets:

[gmail]
credentials_json = '''
{
  "installed": {
    "client_id": "YOUR-CLIENT-ID.apps.googleusercontent.com",
    "project_id": "your-project-id",
    "auth_uri": "https://accounts.google.com/o/oauth2/auth",
    "token_uri": "https://oauth2.googleapis.com/token",
    "auth_provider_x509_cert_url": "https://www.googleapis.com/oauth2/v1/certs",
    "client_secret": "YOUR-CLIENT-SECRET",
    "redirect_uris": ["http://localhost"]
  }
}
'''

8. Click "Deploy"
9. Wait 2-5 minutes for deployment
10. Your app is live! 🎉

STEP 5: FIRST USE
-----------------
1. Open your deployed app URL
2. In sidebar, upload emails_from_excel.txt (or use file uploader)
3. Click "Authenticate with Gmail"
4. Complete OAuth flow
5. Start sending emails!

================================================================================
KEY DIFFERENCES: LOCAL vs CLOUD
================================================================================

LOCAL VERSION:
- credentials.json file in project directory
- token.json file persists on disk
- emails_from_excel.txt in project directory
- run_local_server() for OAuth

CLOUD VERSION:
- credentials.json in Streamlit secrets
- token.json in session state (temporary)
- emails_from_excel.txt uploaded via UI
- OAuth handled through Streamlit interface

================================================================================
FILE UPLOAD FEATURE
================================================================================

The cloud version includes a file uploader in the sidebar:
- Upload emails_from_excel.txt
- File is saved temporarily
- Reloads email database automatically
- No need to commit email file to repo

================================================================================
TOKEN MANAGEMENT
================================================================================

CLOUD CHALLENGE:
- Files don't persist between app restarts
- token.json would be lost

SOLUTION:
- Store token in Streamlit session state
- Token persists during active session
- Re-authenticate if app restarts (takes 30 seconds)

FUTURE IMPROVEMENT:
- Store token in Streamlit secrets (manual update)
- Use external database (Firebase, Supabase)
- Use cloud storage (S3, Google Cloud Storage)

================================================================================
TROUBLESHOOTING
================================================================================

ISSUE: "Credentials not found"
SOLUTION:
- Check Streamlit secrets are correctly formatted
- Verify JSON is valid (no syntax errors)
- Make sure secrets tab has [gmail] section

ISSUE: "OAuth redirect URI mismatch"
SOLUTION:
- Add your Streamlit Cloud URL to Google Cloud Console
- Format: https://your-app-name.streamlit.app
- Wait a few minutes for changes to propagate

ISSUE: "File not found: emails_from_excel.txt"
SOLUTION:
- Use file uploader in sidebar
- Upload emails_from_excel.txt file
- File will be available for that session

ISSUE: "Token expired" or "Not authenticated"
SOLUTION:
- Click "Authenticate with Gmail" again
- Complete OAuth flow
- Token stored in session state

ISSUE: "App sleeps after inactivity"
SOLUTION:
- This is normal for free tier
- Just refresh the page
- App will wake up automatically

ISSUE: "Port already in use"
SOLUTION:
- Not applicable for cloud (handled automatically)
- If deploying elsewhere, use: --server.port=$PORT

================================================================================
SECURITY BEST PRACTICES
================================================================================

✅ DO:
- Use Streamlit secrets for credentials
- Never commit credentials.json to GitHub
- Never commit token.json to GitHub
- Use .gitignore to exclude sensitive files
- Keep OAuth client secret secure
- Regularly rotate credentials

❌ DON'T:
- Commit credentials to version control
- Share Streamlit secrets publicly
- Use production credentials in development
- Store tokens in code
- Hardcode API keys
